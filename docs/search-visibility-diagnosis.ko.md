# 검색 노출 진단 범위

## 목적

이 프로젝트는 사이트를 구현해 배포하거나 운영하려는 사이트 소유자, 개발자, 팀을 위한 검색 노출 진단 서비스다.

주요 목적은 다음 질문에 답하는 것이다.

- 이 사이트가 Google 검색에서 발견, 크롤링, 색인, 노출될 수 있는 상태인가?
- 검색에 노출되지 않는다면 무엇이 노출을 막거나 약화시키고 있는가?
- 검색에 노출되고 있다면 어떤 페이지와 키워드에서 노출이 발생하고 있는가?
- 상위노출 개선 작업을 하기 전에 무엇을 먼저 보강해야 하는가?

이 프로젝트는 일반적인 SEO 점수 측정 도구를 우선 목표로 하지 않는다. 가장 먼저 다루는 문제는 검색 노출 가능성이다. 즉, 사이트가 검색 결과에 나타날 수 있는 상태인지, 그리고 Search Console 데이터상 어떤 증거가 있는지를 진단한다.

순위 개선, 경쟁사 비교, 백링크 권위, 검색결과 클릭 유도 최적화는 2차 목적이다.

## 핵심 진단 모델

SEO 진단 항목은 다음 다섯 그룹으로 나눈다.

| 그룹 | 의미 |
| --- | --- |
| 검색 노출 필수 항목 | 크롤링, 색인, 검색 노출을 막을 수 있는 문제 |
| 검색 노출 데이터 | Google Search Console에서 확인하는 노출수, 클릭수, 검색어, 색인 상태 증거 |
| 상위노출 보강 항목 | 페이지가 크롤링 및 색인 가능한 상태 이후 순위 개선에 도움 되는 항목 |
| SERP 및 CTR 보강 항목 | 검색결과 표시 방식과 클릭률에 영향을 주는 항목 |
| 외부 권위 신호 | 백링크, 브랜드 신호, 제3자 권위 데이터 |

## 구현 가능성 라벨

각 진단 항목에는 구현 가능성 라벨을 붙인다.

| 라벨 | 의미 |
| --- | --- |
| URL 자동 가능 | 제출된 URL을 크롤링하거나, 헤더를 가져오거나, HTML을 파싱하거나, 연결 리소스를 확인해 검사 가능 |
| 무료 API 가능 | 무료 공개 API로 검사 가능. 보통 API key 또는 쿼터 제한이 있음 |
| GSC 연동 필요 | 사용자가 OAuth로 Google Search Console property를 연결해야 검사 가능 |
| 부분 가능 | 휴리스틱으로 추정 가능하지만 확정적인 통과/실패 판정으로 다루면 안 됨 |
| 결제 필요 | 유료 SEO, SERP, 백링크, 로컬 데이터 제공자가 필요 |
| 사람 판단 필요 | 주관적 검토, 비즈니스 맥락, 편집 판단이 필요 |

## 1순위: 검색 노출 필수 항목

이 항목들은 사이트가 크롤링, 색인, 검색 노출 대상이 될 수 있는지를 결정한다.

| 항목 | 목적 | 구현 가능성 | 우선순위 |
| --- | --- | --- | --- |
| HTTP 200 상태 | 페이지가 정상 접근 가능한지 확인 | URL 자동 가능 | 필수 |
| 3xx 리디렉션 체인 | 과도한 리디렉션, 루프, 불안정한 canonical 경로 탐지 | URL 자동 가능 | 필수 |
| 404 및 5xx 오류 | 색인될 수 없는 페이지 탐지 | URL 자동 가능 | 필수 |
| HTTPS 사용 가능 여부 | 보안 접근 및 canonical 프로토콜 확인 | URL 자동 가능 | 필수 |
| HTTP에서 HTTPS 리디렉션 | 비보안 접근이 안전하게 정리되는지 확인 | URL 자동 가능 | 중요 |
| robots.txt 존재 여부 | 크롤러 정책 파일 존재 확인 | URL 자동 가능 | 필수 |
| robots.txt 차단 여부 | 제출 URL 또는 주요 경로가 차단되는지 탐지 | URL 자동 가능 | 필수 |
| meta robots noindex | 페이지 단위 색인 차단 탐지 | URL 자동 가능 | 필수 |
| meta robots nofollow | 페이지 단위 링크 추적 차단 탐지 | URL 자동 가능 | 중요 |
| X-Robots-Tag noindex | 헤더 기반 색인 차단 탐지 | URL 자동 가능 | 필수 |
| canonical 태그 존재 여부 | 선언된 대표 URL 확인 | URL 자동 가능 | 필수 |
| canonical 대상 접근 가능 여부 | canonical URL이 정상 응답하는지 확인 | URL 자동 가능 | 필수 |
| canonical이 다른 페이지를 가리킴 | self-canonical 누락 또는 URL 통합 문제 후보 탐지 | URL 자동 가능 | 중요 |
| sitemap 존재 여부 | 검색엔진의 URL 발견 보조 | URL 자동 가능 | 중요 |
| robots.txt 내 sitemap 선언 | sitemap 발견성 개선 여부 확인 | URL 자동 가능 | 중요 |
| sitemap URL 접근 가능 여부 | sitemap을 가져올 수 있는지 확인 | URL 자동 가능 | 중요 |
| sitemap 내 오류 URL | sitemap 안의 404, 5xx, 리디렉션 URL 탐지 | URL 자동 가능 | 중요 |
| sitemap 최신성 | `lastmod` 존재 및 타당성 확인 | URL 자동 가능 | 중요 |
| 과도한 JavaScript 렌더링 의존 | 단순 크롤러가 콘텐츠를 보지 못할 가능성 탐지 | 부분 가능 | 필수 |
| Google 실제 색인 상태 | Google이 해당 URL을 색인했는지 확인 | GSC 연동 필요 | 필수 |
| Google 선택 canonical | Google이 어떤 URL을 대표 URL로 보는지 확인 | GSC 연동 필요 | 필수 |
| Google crawl 상태 | Google의 마지막 crawl 및 접근 결과 확인 | GSC 연동 필요 | 중요 |
| soft 404 분류 | Google이 낮은 가치 또는 오류성 페이지로 보는지 탐지 | GSC 연동 필요 | 중요 |
| 중복 색인 페이지 | 색인 중복 또는 canonical 충돌 탐지 | GSC 연동 필요 + 크롤링 비교 | 중요 |

## 2순위: 검색 노출 및 키워드 진단

이 항목들은 제품 목적의 핵심이다. 제출된 URL만으로는 신뢰성 있게 얻을 수 없으며, 사용자가 자신의 Google Search Console property를 연결해야 한다.

| 항목 | 목적 | 구현 가능성 | 우선순위 |
| --- | --- | --- | --- |
| Google 노출 발생 여부 | 사이트가 Google 검색에 노출되고 있는지 판단 | GSC 연동 필요 | 필수 |
| 검색어 목록 | 사이트를 노출시키는 키워드 식별 | GSC 연동 필요 | 필수 |
| 검색어별 노출수 | 키워드별 검색 노출 규모 측정 | GSC 연동 필요 | 필수 |
| 검색어별 클릭수 | 키워드별 실제 검색 유입 측정 | GSC 연동 필요 | 중요 |
| 검색어별 CTR | 노출은 되지만 클릭되지 않는 키워드 탐지 | GSC 연동 필요 | 중요 |
| 검색어별 평균 순위 | 순위 개선 전 현재 검색 위치 파악 | GSC 연동 필요 | 중요 |
| 페이지별 노출수 | 어떤 페이지가 노출되는지 식별 | GSC 연동 필요 | 필수 |
| 페이지별 클릭수 | 어떤 페이지가 검색 유입을 받는지 식별 | GSC 연동 필요 | 중요 |
| 페이지-검색어 매핑 | 어떤 페이지가 어떤 검색어와 연결되는지 이해 | GSC 연동 필요 | 필수 |
| 국가별 노출 | 지역별 검색 노출 진단 | GSC 연동 필요 | 중요 |
| 기기별 노출 | 모바일/데스크톱 노출 차이 진단 | GSC 연동 필요 | 중요 |
| 날짜별 추세 | 노출 하락, 개선, 색인 변화 탐지 | GSC 연동 필요 | 중요 |
| 색인됐지만 노출 없는 페이지 | Google에 존재하지만 검색에 보이지 않는 페이지 탐지 | GSC 연동 필요 + sitemap/크롤링 데이터 | 중요 |
| 노출은 있지만 클릭 없는 페이지 | title, snippet, 검색 의도, 순위 문제 후보 탐지 | GSC 연동 필요 + HTML 분석 | 중요 |

| 평균 순위 8~20위 검색어 | 개선 여지가 큰 근접 기회 키워드 탐지 | GSC 연동 필요 | 상위노출 보강 |

## Google Search Console 연동

이 프로젝트는 검색 노출 데이터를 위해 사용자가 승인하는 Google Search Console 연동을 지원해야 한다.

사용자 흐름은 다음과 같다.

1. 사용자가 서비스에 로그인한다.
2. 사용자가 "Google Search Console 연결"을 선택한다.
3. 서비스가 Google OAuth 접근 권한을 요청한다.
4. 사용자가 Search Console 읽기 권한을 승인한다.
5. 서비스가 해당 Google 계정에서 접근 가능한 Search Console property 목록을 보여준다.
6. 사용자가 진단할 property를 선택한다.
7. 서비스가 크롤링 기반 검사와 Search Console inspection 및 검색 성과 데이터를 결합한다.

권장 읽기 전용 OAuth scope:

```text
https://www.googleapis.com/auth/webmasters.readonly
```

더 넓은 쓰기 가능 scope는 제품이 명시적으로 sitemap 제출 또는 Search Console 상태 변경을 제공할 때만 사용한다.

```text
https://www.googleapis.com/auth/webmasters
```

중요 제약:

- 서비스는 연결된 사용자가 Search Console에서 접근 권한을 가진 property만 검사할 수 있다.
- Search Console을 통해 임의의 경쟁사 사이트를 검사할 수 없다.
- 공개 출시 시 Google OAuth consent screen 설정, 앱 검증, 개인정보처리방침, 승인된 도메인, 안전한 token 처리가 필요할 수 있다.
- refresh token은 암호화해 저장하고 서버 측에서만 사용해야 한다.

## 제품 포지셔닝

제품은 진단 결과를 다음 순서로 보여줘야 한다.

1. Google이 사이트에 접근할 수 있는가?
2. 색인을 명시적으로 막는 요소가 있는가?
3. Google이 실제로 사이트를 색인했는가?
4. 사이트가 검색 노출을 받고 있는가?
5. 어떤 키워드와 페이지에서 노출이 발생하고 있는가?
6. 색인됐지만 노출되지 않는 페이지는 무엇인가?
7. 노출됐지만 클릭되지 않는 페이지는 무엇인가?
8. 어떤 기술, 콘텐츠, 구조 항목을 보강해야 하는가?
9. 어떤 순위/권위 항목이 외부 유료 데이터 또는 전문가 검토를 필요로 하는가?

이 순서는 진단의 목적을 선명하게 유지한다. 검색 노출이 먼저이고, 순위 개선은 그다음이다.

## 초기 분류의 비목표

다음 영역은 핵심 검색 노출 체크로 다루지 않는다.

- 색인의 전제조건으로서의 백링크 권위
- 노출의 전제조건으로서의 도메인 권위 점수
- 노출 진단의 필수 조건으로서의 경쟁사 SERP 비교
- 핵심 검색 노출 이슈로서의 소셜 공유 메타데이터
- 노출 영향 설명이 없는 일반 SEO 점수 합산

## 권장 결과 심각도

각 finding은 검색 노출 차단 여부를 기준으로 심각도를 매긴다.

| 심각도 | 의미 |
| --- | --- |
| Critical | 크롤링, 색인, 검색 노출을 막을 수 있음 |
| High | 발견성, 색인 품질, 검색어 매칭에 강한 영향을 줌 |
| Medium | 순위, CTR, 구조, 기술 품질을 보조함 |
| Low | 있으면 좋은 항목, 소셜 표시, polish, 2차 최적화 |
| Unavailable | 결제, 사용자 승인, 사람 판단이 필요함 |

## 요약

이 프로젝트는 순위 개선보다 검색 노출을 먼저 진단해야 한다. 가장 중요한 신호는 크롤링 가능성, 색인 가능성, Search Console의 색인 증거, Search Console의 검색어 노출 데이터다.

URL만으로도 많은 기술적 차단 요소를 찾을 수 있지만, 어떤 키워드에서 사이트가 노출되는지는 증명할 수 없다. 키워드 노출 데이터는 해당 property의 소유자 또는 접근 권한을 가진 사용자의 Google Search Console 연결이 필요하다.

유료 SEO API는 백링크와 경쟁 순위 분석에 유용하지만, 핵심 검색 노출 진단과는 명확히 분리해야 한다.
